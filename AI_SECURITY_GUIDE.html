🔐 Protected Files & Patterns

Environment Files

  • .env - Default environment variables
  • .env.local - Local overrides
  • .env.*.local - Environment-specific (dev, prod, test)

Configuration Files

  • appsettings.json - Default config
  • appsettings.*.json - Environment variants
  • appsettings.Development.json
  • appsettings.Production.json

Secret Directories

  • secrets/ - Any files inside
  • .secrets/ - Any files inside

Private Keys & Certificates

  • *.pem - PEM certificates
  • *.key - Private keys
  • *.pfx - PKCS#12 certificates
  • *.p12 - PKCS#12 format
  • id_rsa* - SSH keys
  • id_ed25519* - Ed25519 SSH keys

Files Matching Sensitive Patterns

  • Any file with "secret" in the name
  • Any file with "credentials" in the name
  • Any file with "password" in the name
  • Any file with "api_key" or "apikey" in the name
  • Any file with "token" in the name

🔧 Technical Details

Claude Code Security

Uses permission deny rules in .claude/settings.json:

{ "permissions": { "deny": [ "Read(.env)", "Read(.env.*.local)", "Read(appsettings.json)", "Edit(.env)", "Write(.env)", ... ] } }

OpenCode Security

Uses structured permission blocking in opencode.json:

{ "permission": { "read": { ".env": "deny", ".env.local": "deny", "appsettings.json": "deny", "**/secrets/**": "deny" }, "edit": { ".env": "deny", "appsettings.json": "deny", "**/secrets/**": "deny" } } }

Zed AI Security

Multi-layered permission system in .zed/settings.json:

  • Deny rules: Explicit permission blocking for Read, Write, Edit operations
  • Security flags: Boolean toggles for environment files, configs, and secrets directories
  • Key pattern matching: Blocks private keys (*.pem, *.key, *.pfx, *.p12, *.jks, id_rsa*, id_ed25519*, etc.)

Gemini Security

Dual-layer approach using file filtering and environment variable redaction:

  • .geminiignore: Mirrors .claude/settings.json patterns to exclude sensitive files from context
  • .gemini/settings.json: Enables environment variable redaction to mask sensitive values
  • File filtering: Respects both .geminiignore and .gitignore patterns automatically

GitHub Copilot Custom Instructions

Behavioral guardrails defined in .copilot-instructions

  • Copilot reads this file and follows the guidelines
  • Provides education and warnings
  • Guides secure patterns

Git Protection

Comprehensive patterns in .gitignore covering:

  • All environment variable files
  • Configuration files with secrets
  • Private keys and certificates
  • Credential/token/password patterns

❓ FAQ & Support

Q: What if I need to work with secret files?

A: Handle them manually outside of AI assistants. Use your code editor, terminal, or configuration management tools directly.

Q: Can I disable these restrictions?

A: Project-level restrictions apply to everyone. Individual users can add overrides in .claude/settings.local.json (not committed), but this is not recommended for security reasons.

Q: What if Claude Code shows an error about permissions?

A: This is working as intended. It means you're trying to access a protected secret file. This is a good sign that the protection is active.

Q: Does this slow down development?

A: No. The restrictions are transparent. You simply can't ask AI about secrets, which is the desired behavior. Use environment variables and configuration patterns instead.

Q: How do I report a security issue?

A: If you accidentally expose a secret or find a way to bypass these protections, report it to your security team immediately.
